Lesson-1-2: HTML

HTML stands for Hypertext Markup Language. HTML documents form the majority of the content on the web.

There are approximately 30 billion pages within the web. A variety of document types and hyperlinks exist. These include:

  • Text, the content (what you see).
  • Markup, what the content looks like.
  • URLs, references to other documents, i.e. images and videos.
  • Links to other pages.

Udacity offers this video providing an overview.

Lesson-1-3: Tags and Elements

HTML documents are composed of HTML elements. When writing HTML, the user/programmer defines he type of HTML elements appearing within the browser by using HTML tags. Udacity's video explains the distinctions.

Tags are primarily composed of opening and closing tags. However, there are 'void' tags which do not require a closing tag.

Failing to properly close a tag may result in everything following the opening tag to assume that attribute of that tag. For example, if one were to use an opening tag of "em" and fail to close that tag, everything from the opening tag forward would be shown in talics.

Sample tags include:

  • <b> for bolding text between opening and closing tags
  • <em> for italicizing text
  • <br> creates a break between text.
  • <p> paragraph, does just that. LOL. It has content.

Attributes or properties of HTML, used properly, create powerful pages. Attributes include:

  • HREF, which creates a link to access additional information/site. For example: Udacity
  • <img> which does not contain a closing tag. It is considered a "void" tag. In the example below the use of 'alt' creates an alternate view of the referenced source. This is quite helpful should a web error occur and supports a blind computer user.
    For example: <img src="URL" alt=text>

Anchor tags use the "href" attribute while image tags use the "src" attribute.

Computers have yet to reach a full state of artificial intelligence and currently possess the properties applied by programmers/web developers.

Lesson-1-5: Inline vs Block Elements

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside the box.

Inline elements include:

  • <a> used to create links such as UdacityOpening and closing tags used.
  • <br> used toens a line and subsequently wraps text to next line. No closing tag.
  • <em> emphasizes and italicizes words. Opening and closing tags used.
  • <img> introduces a picture. No closing tag
  • <span> uses opening and closing tags
  • <strong> uses opening and closing tags

Block elements include:

  • <p> which creates in invisible "block" around everything included within opening and closing tags. Additional attributes such as height and width may also be applied.
  • <form> and its closing tags where developer is creating a form for a user to complete.

Additionally these block elements may be considered 'container' elements and have content.

  • <div> and its closing tag. Syntax: <div class=”bar”>text </div>
  • <span> and its closing tag. Syntax: <span class =”foo”>text </span> where 'class’ is a CSS style.

Lesson 3: Adding CSS Style to HTML Structure

Lesson-3-1: Cascading Style Sheets (CSS)

CSS stands for Cascading Style Sheets CSS provide programmers a way to control the style of related HTML elements. This is done by giving similar HTML elements the same class name and then specifying the applied styles to each class.

Lesson-3-2: Benefits of Style Sheets

Avoiding repetition: Create cleaner code which may then become 'reuseable'. "Work smarter, not harder".

Creating consistency: Use CSS (Cascading Style Sheets) to standardize the look of web pages.
